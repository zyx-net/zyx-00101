# 门店交接班异常追踪系统

便利店早中晚班交接班管理 Web 应用，用于登记交接班异常（现金差额、库存短缺），接班人确认，店长复核关闭或退回，全流程操作历史记录和数据持久化到本地 JSON 文件，重启不丢失。

## 技术栈

- 后端：Node.js + Express
- 前端：原生 HTML/CSS/JavaScript（无构建工具）
- 存储：本地 JSON 文件

## 启动说明

```bash
# 1. 安装依赖
npm install

# 2. 启动服务
npm start
```

启动后访问：**http://localhost:3000**（端口被占用时可用 `PORT=3001 npm start` 指定其他端口）

## 主流程操作步骤

以下步骤可直接复现完整流程：

### 第一步：员工（王早班）创建班次并提交交接

1. 使用 `staff1 / staff123` 登录
2. 点击「+ 创建班次」
3. 填写：
   - 门店：便利店-朝阳店
   - 班次类型：早班
   - 交接日期：今天
   - 交班人：王早班
   - 接班人：赵中班
   - 勾选交接清单（必填项必须勾选）
   - 备注：早班交接一切正常
4. 点击「创建班次」
5. 在班次列表中找到该班次，点击「详情」
6. 点击「提交交接」→ 状态变为「待确认」

### 第二步：员工（赵中班）确认交接并登记异常

1. 使用 `staff2 / staff123` 登录
2. 进入班次详情
3. 点击「确认交接」→ 状态变为「已确认」
4. 点击「+ 登记异常」
   - 类型：现金差额
   - 金额：-50
   - 责任人：王早班
   - 描述：收银机比系统少 50 元
5. 再登记一条：
   - 类型：库存短缺
   - 品项：可口可乐
   - 数量：3
   - 责任人：王早班
6. 点击「提交复核」→ 状态变为「复核中」

### 第三步：店长（张店长）处理异常并关闭班次

1. 使用 `manager1 / manager123` 登录
2. 进入班次详情，查看两条异常
3. 点击每条异常的「处理」→ 填写处理说明后「确认处理」
4. 点击每条异常的「关闭」→ 填写关闭意见后「确认关闭」
5. 所有异常关闭后，点击「关闭班次」
6. 填写复核意见，确认关闭
7. 状态变为「已关闭」

### 第四步：查看操作历史与导出

1. 左侧菜单「操作历史」可查看所有操作时间线
2. 班次管理 / 异常管理 页面顶部可按门店、日期筛选后，点击「导出 CSV」或「导出 JSON」下载文件

## 权限与错误提示（会给出明确错误，不会破坏原状态）

- **无权关闭**：员工点击「关闭班次」按钮将报错「权限不足：仅店长可执行此操作」
- **重复确认**：已确认班次再次确认将报错「班次已确认，不可重复确认」
- **未处理异常关闭**：存在未关闭异常时关闭班次将报错「存在 N 条未关闭异常，需先处理后再关闭班次」
- **非本门店操作**：跨门店操作将报错「仅本门店店长可关闭班次」
- **交班人接班人相同**：创建班次时报错「交班人与接班人不能为同一人」

## 本地配置

- `config/stores.json`：门店信息，可新增/修改/删除门店
- `config/checklist.json`：常用交接清单项，支持必填/可选
- `config/users.json`：用户账号（密码明文，仅本地演示用）
- `data/shifts.json`：班次数据（运行期生成）
- `data/exceptions.json`：异常数据
- `data/history.json`：操作历史

修改 `config/` 下的文件后重启服务生效。
